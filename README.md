# DevTools Workspace — Free Online JSON & Developer Data Tools Platform

**Free, private, client-side developer tools.** Format, validate, minify, and convert JSON, YAML, XML, and more. Generate code from JSON (TypeScript, Python, Java, Go, and 6 more). All processing runs in your browser; nothing is sent to any server.

🌐 **Live site:** [formatterjson.org](https://formatterjson.org)

---

## Features

- **100% client-side** — Your data never leaves your device. No server uploads.
- **Platform homepage** — Dedicated landing at `/` with hub pages by category.
- **Monaco editor** — Syntax highlighting, line numbers, larger font size, and keyboard shortcuts.
- **Editor actions** — Share link (copy URL with encoded input), print data only (input + output in a new window), search in both input and output panels (Monaco find with closeable widget), per-panel fullscreen (input or output fullscreen with exit).
- **Dark / light theme** — System preference or manual toggle.
- **Multi-convert bar** — From JSON: convert to CSV, YAML, or XML in one click.
- **Keyboard shortcuts** — Run (Ctrl+Enter), minify (Ctrl+Shift+M), toggle sidebar (Ctrl+B), shortcuts help (?).
- **Static export** — Next.js static site; easy to host anywhere (Vercel, Netlify, GitHub Pages).

## Tech Stack

- **Framework:** [Next.js 14](https://nextjs.org/) (App Router, static export)
- **Language:** TypeScript
- **State:** [Zustand](https://github.com/pmndrs/zustand) with persist (localStorage)
- **Editor:** [Monaco Editor](https://microsoft.github.io/monaco-editor/) (VS Code editor in the browser)
- **Styling:** [Tailwind CSS](https://tailwindcss.com/) with custom `dt-*` theme palette
- **Data:** [js-yaml](https://github.com/nodeca/js-yaml), native JSON/XML; no backend

## Getting Started

### Prerequisites

- Node.js 18+
- npm (or yarn / pnpm)

### Install and run

```bash
git clone https://github.com/khadkeshwarkp/formatterjson1.git
cd formatterjson1
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). The app runs as a static-friendly Next.js app with client-side routing.

### Build for production

```bash
npm run build
```

Output is in the `out/` directory (static export). Serve with any static host:

```bash
npm start
# or: npx serve out
```

### Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start dev server (localhost:3000) |
| `npm run build` | Static export to `out/` |
| `npm run start` | Serve production build |
| `npm run lint` | Run ESLint |

## Project structure (high level)

- `app/` — Next.js App Router pages. Homepage at `/`, hub pages (`json-tools`, `xml-tools`, etc.), each tool at `app/<tool-id>/page.tsx`.
- `components/` — `PlatformHomePage` (homepage), `HubPage` (category hubs), `ToolPage`, `ToolWorkspace`, `MonacoWrapper`, `OutputPanel`, `WorkspaceShell`, `Sidebar`, SEO components.
- `lib/` — `tools-registry.ts` (tool metadata), `processors.ts` (transform functions), `json-to-code.ts` (JSON-to-language generators), `hub-content.ts` (hub copy and FAQs), `hubs.ts` (category→hub mapping), `store.ts` (Zustand).
- `styles/` — Global CSS and Tailwind.

New tools: (1) add `ToolMeta` in `lib/tools-registry.ts`, (2) add processor in `lib/processors.ts`, (3) register in `ToolWorkspace.tsx`, (4) add `app/<tool-id>/page.tsx`. See [AGENTS.md](./AGENTS.md) and [RESTRUCTURE.md](./RESTRUCTURE.md) for full architecture.

## SEO and privacy

- **SEO:** Each tool and hub page has unique meta title, description, and FAQ JSON-LD. WebSite, Breadcrumb, CollectionPage, and WebApplication schemas where applicable. Sitemap includes homepage, hubs, tools, and legal pages.
- **Privacy:** No analytics or tracking until consent. No server-side processing; all tool logic runs in the browser.
- **Hosting:** Suitable for [formatterjson.org](https://formatterjson.org), Vercel, Netlify, or any static host.
